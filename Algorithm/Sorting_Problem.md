# Sorting Problem

```sh

# [입력]

n개의 숫자들의 배열 <a1, a2, a3, ... , an>

# [출력]

입력된 숫자의 배열이 조건을 만족하도록 다시 나열한 결과 값

# [Example]

[input] : <5, 2, 4 ,6, 1, 3>

[output] : <1, 2, 3, 4, 5, 6>

```

## Selection sort(선택 정렬)

- 최소값 선택 정렬 : 가장 작은 값을 선택(오름차순)

1. 정렬되지 않은 숫자 중에 가장 작은 숫자를 선택

2. 선택한 숫자를 정렬되지 않은 숫자들 중 첫 번째 숫자와 자리를 바꾸면 선택된 숫자는 정렬된 것

3. 모든 숫자를 옮길 때 까지 1-2번의 과정을 반복

- 최대값 선택 정렬 : 가장 큰 값을 선택(내림차순)

```sh

# 최소값 정렬 ex


[input] : <5, 2, 4, 6, 1, 3>

# 가장 작은 값인 1을 선택하고 5와 change

<1, 2, 4, 6, 5, 3>


# 남은 것중 가장 작은 값을 또 다시 선택(2를 선택)

<1, 2, 4, 6, 5, 3>

# 2는 그대로 내려온다. 이와 같이 배열의 마지막 까지 반복


# 최대값 정렬도 동일한 방식

```

### Selection Sort의 정확성 증명

- 수학적 귀납법 이용

- i번째 선택한 숫자가 i번째로 작은 or 큰 숫자인지를 증명


### Selection Sort의 성능 분석

- 최선/최악의 경우 수행시간 : (n 제곱)

- 최선/최악의 경우 공간 : (n)
